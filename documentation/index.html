Troubleshooting Common Issues

Store Connection Issues

Cannot Connect to Shopify Store

The plugin is unable to establish a connection with your Shopify store.

Common Causes & Solutions:
  • Invalid Store URL: Ensure your store URL follows the format: yourstore.myshopify.com
  • Invalid API Token: Verify your Shopify API access token is correct and hasn't expired
  • Insufficient Permissions: Your API token must have read access to products, customers, orders, and other data you want to import
  • Network Issues: Check your server's internet connection and firewall settings
  • Shopify API Limits: Ensure you haven't exceeded Shopify's API rate limits

Store Shows as "Inactive"

Your store connection appears but shows an inactive status.

Solutions:
  • Click the "Test" button to verify the connection
  • Check if your Shopify store is still active and accessible
  • Verify your API token hasn't been revoked or changed
  • Try editing the store and re-entering the credentials

Import Process Issues

Import Fails or Times Out

The import process stops unexpectedly or takes too long to complete.

Solutions:
  • Reduce Batch Size: Lower the number of items processed per batch (try 25 instead of 100)
  • Check Server Resources: Ensure your server has sufficient memory and processing power
  • Increase PHP Limits: Modify max_execution_time and memory_limit in your PHP configuration
  • Verify Data Integrity: Check if there are corrupted or invalid records in your Shopify data
  • Check Logs: Review the import logs for specific error messages

Slow Import Performance

Imports are taking much longer than expected.

Performance Optimization:
  • Optimize Server: Use a dedicated server or VPS instead of shared hosting
  • Enable Caching: Install and configure a caching plugin like WP Rocket or W3 Total Cache
  • Database Optimization: Clean up your WordPress database and optimize tables
  • Reduce Concurrent Users: Limit other processes running on your server during imports
  • Use Smaller Batches: Process data in smaller chunks to avoid timeouts

Partial Import Success

Some items import successfully while others fail.

Common Causes:
  • Duplicate Products: Products with existing SKUs or titles may be skipped
  • Invalid Data: Products with missing required fields (title, price) will fail
  • Media Issues: Large images or corrupted media files may cause failures
  • Permission Issues: WordPress may not have permission to create media files

Data Import Issues

Products Not Importing

Products are not appearing in your WooCommerce store after import.

Troubleshooting Steps:
  • Check if products are being imported as drafts (check Drafts in WooCommerce)
  • Verify your WooCommerce product catalog settings
  • Check if products are assigned to the correct categories
  • Review import logs for specific error messages
  • Ensure WooCommerce is properly configured and activated

Images Not Importing

Product images are missing after the import process.

Solutions:
  • Check WordPress media library permissions
  • Ensure your server can download external images
  • Verify image URLs are accessible from your server
  • Check if images are being blocked by firewall or security plugins
  • Try importing images separately in a smaller batch

Customer Data Issues

Customer information is not importing correctly.

Common Issues:
  • Email Conflicts: Customers with existing email addresses may not be imported
  • Invalid Addresses: Customers with malformed address data may be skipped
  • Role Mapping: Check if customer roles are being assigned correctly
  • Password Issues: Imported customers may need to reset their passwords

Need More Help? If you're still experiencing issues after trying these solutions, check the import logs for detailed error messages. The logs will provide specific information about what went wrong and help identify the root cause of the problem.

Frequently Asked Questions

What is WP Migrate & Import Shopify to WooCommerce?

WP Migrate & Import Shopify to WooCommerce is a professional WordPress plugin that allows you to seamlessly migrate your entire Shopify store data to WooCommerce, including products, customers, orders, and content.

What data can be migrated from Shopify to WooCommerce?

The plugin can migrate products (with images, variants, and SEO data), customers (with accounts and addresses), orders (with line items and status), blog posts, pages, and coupons from your Shopify store to WooCommerce.

Is the migration process safe?

Yes, the migration process is designed to be safe and non-destructive. We recommend testing the migration on a staging environment first, and the plugin includes comprehensive logging and monitoring features.

What are the system requirements?

The plugin requires WordPress 5.0 or higher, WooCommerce 3.0 or higher, PHP 7.4 or higher, and sufficient server resources to handle the migration process. Detailed requirements are listed in the System Requirements section.

How long does the migration take?

Migration time depends on the amount of data being transferred. Small stores with few products may complete in minutes, while larger stores with thousands of products may take several hours. The plugin provides real-time progress monitoring.

How do I get my Shopify API access token?

Follow the "Finding Shopify Details" section above. You'll need to create a private app in your Shopify admin, grant the required permissions, and copy the access token from the API credentials tab.

What permissions does my Shopify private app need?

At minimum, your app should have read access to Products, Customers, Orders, Content, and Discounts. See the "Finding Shopify Details" section for a full list.

Why am I getting an "Invalid API token" error?

Double-check that you copied the entire access token and that it hasn't expired or been regenerated. Also, ensure your app has the correct permissions.

Can I import multiple Shopify stores?

Yes! The plugin supports connecting and managing multiple Shopify stores. Use the "Add Store" section for each new store you want to import.

What data can I import from Shopify?

You can import products, customers, orders, coupons, and more. See the "Import Configuration" section for details on what's supported.

How often does the plugin sync data?

By default, data is imported when you run an import job. If you want automatic syncing, check the plugin settings for scheduling options.

Will importing overwrite my existing WooCommerce data?

No, the importer creates new entries and does not overwrite existing WooCommerce products, customers, or orders unless you specifically map them to existing records.

How do I update the plugin?

Update the plugin like any other WordPress plugin‚Äîvia the Plugins page in your WordPress admin. Always back up your site before updating.

Where can I get support?

If you need help, visit our support page (link), email us at [support@example.com], or check the "Support & Resources" section above.

What should I do if the import fails or is incomplete?

Check the import logs for error messages. Common issues include invalid credentials, missing permissions, or server timeouts. Try re-running the import or contact support if the problem persists.
